Detailed Description

Virtual class Parent of CPU_FE and GPU_FE classes.

Classes CPU_FE and GPU_FE are the front-end classes for CPU and GPU versions of the algorithm respectively. Both these classes inherit MAP_FE, which is a virtual class: meaning an object of this class cannot be created. But such an implementation ensures two things:

  1. Any implementation of any mapping algorithm must neccessarily implement the member methods of MAP_FE.
  2. Other classes dependent on the global map need not change their implementation depending on the algorithm used as long as the front end of the implementation is a child of Map_FE. Also pointer of a child class can be cast to their parent class.
